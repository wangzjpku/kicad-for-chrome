# Plan 3: AI 智能项目创建对话框

## 项目概述
在新建项目时增加 AI 智能对话框功能，用户可以通过自然语言描述项目需求，AI 自动生成电路原理图和 PCB。

## 核心功能需求

### 1. AI 对话框界面
- 模态对话框，包含富文本输入区域
- 支持自然语言输入项目需求
- 显示 AI 生成进度和状态
- 支持方案预览和编辑

### 2. AI 需求分析引擎
- 解析用户输入的自然语言
- 提取关键技术参数（电压、电流、功率等）
- 识别功能模块和性能要求
- 生成结构化需求文档

### 3. 项目方案生成
- 器件选型建议（型号、封装、采购）
- 电路原理概述
- 技术参数表
-  BOM 清单预生成

### 4. 原理图自动生成
- 生成 KiCad 可解析的 JSON 指令
- 器件列表（名称、型号、封装、数量）
- 连线关系（引脚连接规则）
- 布局规则（核心器件位置、信号走向）
- 标注要求（参数标注、网络名、注释）

### 5. PCB 布局生成（可选扩展）
- 基于原理图自动布局
- 器件位置建议
- 布线规则生成

## 技术架构

### 前端组件
- AIProjectDialog.tsx - AI 对话框组件
- ProjectSpecViewer.tsx - 方案预览组件
- SchematicPreview.tsx - 原理图预览组件

### 后端 API
- /api/v1/ai/analyze - 需求分析
- /api/v1/ai/generate-schematic - 生成原理图指令
- /api/v1/ai/generate-pcb - 生成 PCB 布局指令
- /api/v1/ai/validate - 方案验证

### AI 集成
- 支持多种 AI 模型（Claude/GPT）
- 系统提示词模板（对话框模板.txt）
- JSON 格式输出规范

## 实施阶段

### Phase 1: UI 对话框
- 设计 AI 对话框 UI
- 需求输入界面
- 进度显示

### Phase 2: AI 后端集成
- API 路由开发
- AI 服务封装
- 系统提示词配置

### Phase 3: 原理图生成
- 需求解析引擎
- 器件库匹配
- JSON 指令生成

### Phase 4: 前端渲染
- 方案预览
- 原理图渲染
- 编辑和确认

## 依赖项
- Claude API / OpenAI API
- 器件库数据
- 现有原理图/PCB 编辑器

## 风险与挑战
1. AI 生成准确性需要验证
2. 器件库覆盖范围
3. 复杂电路的自动布局效果
