## 1. 项目概述与测试范围

### 1.1 项目核心功能

本项目是一个基于浏览器的 KiCad EDA 软件控制系统，核心功能是将用户的自然语言需求转换为电路原理图和 PCB 设计。

| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| **AI 语义分析** | 将用户自然语言需求解析为电路方案 | P0 |
| **原理图生成** | 生成 sch 原理图文件 | P0 |
| **PCB 设计** | 生成 PCB 布局和走线 | P1 |
| **封装库管理** | 自动匹配元件封装 | P0 |
| **DRC 检查** | 设计规则检查和验证 | P1 |
| **文件导出** | Gerber/BOM/STEP 导出 | P1 |

## 2. Ralph Loop 测试迭代框架

Ralph Loop 是一种快速迭代测试方法论，核心原则：

1. **快速反馈** - 每个迭代周期短 (1-2天)
2. **自动化优先** - 自动化测试覆盖 80%+
3. **持续集成** - 每次提交触发测试
4. **数据驱动** - 基于测试结果调整策略

### 3.2 用户操作步骤详情

#### 步骤 1: 启动与连接
1. 用户打开浏览器访问 `http://localhost:3000`
2. 前端加载，尝试连接后端 WebSocket
3. 后端返回连接状态（前/后端连接指示器）
4. 用户确认 "后端服务: 已连接" 状态

#### 步骤 2: 新建项目
1. 点击 "新建项目" 或 "AI 创建" 按钮
2. 弹出项目创建对话框
3. 输入项目名称和描述
4. 点击确认创建项目

#### 步骤 3: 输入需求（自然语言）
1. 点击 "AI 生成" 按钮打开AI对话框
2. 在文本框中输入电路需求
3. 可选：选择电路类型/复杂度
4. 点击 "生成" 按钮提交

#### 步骤 4: AI 处理与生成
1. 后端接收需求文本
2. AI 模块解析需求，提取：
   - 元件类型和数量
   - 参数规格
   - 封装要求
3. 封装推荐模块匹配封装
4. 生成原理图数据结构
5. 返回前端显示

#### 步骤 5: 原理图查看与编辑
1. 前端显示生成的原理图
2. 用户可选择/移动元件
3. 可手动添加/删除元件
4. 可编辑元件属性

#### 步骤 6: PCB 设计
1. 从原理图创建 PCB
2. 自动布局元件
3. 自动/手动走线
4. 进行 DRC 检查

#### 步骤 7: 导出文件
1. 点击 "导出" 菜单
2. 选择导出格式（Gerber/BOM/STEP）
3. 选择保存路径
4. 执行导出
5. 下载/保存文件
